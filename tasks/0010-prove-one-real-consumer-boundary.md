## Intended result

The public `kapsel.cloud` site would render an existing bounded explanation from an attributable
browser artifact generated at one exact clean Grafik revision. The consumer would invoke the
existing scene and measured-trace interfaces while owning product-data mapping, DOM/SVG/CSS, browser
measurement, accessibility, responsive behavior, and reduced-motion consumption.

This packet proves or falsifies the documented consumer boundary before Grafik expands its editor
surface. It does not make the consumer a Grafik build input or turn one integration into a stable
compatibility promise.

## Constraints

- No source-level consumer dependency, sibling path, symlink, submodule, or runtime network
  dependency.
- No product-specific record parser, schema, style, copy, or semantic result meaning in Grafik.
- No DOM access, selectors, markup, literal brand colors, or browser authority in Rust.
- No editor work, package extraction, generic plugin seam, or second consumer abstraction.
- Grafik's default gate remains deterministic and independent of the consumer checkout.
- Consumer failure to load or run the decorative explanation must not block its readable content or
  primary action.

## Work

1. Select one clean Grafik revision and produce its attributable browser artifact.
2. Integrate the artifact through a reviewed `kapsel.cloud` change without modifying or automating
   writes into that checkout from Grafik.
3. Map one existing bounded explanation into the current scene request, render its consumer-owned
   structure, return measured geometry, and consume the complete trace.
4. Preserve readable no-JavaScript and reduced-motion states, then test responsive behavior and
   failure fallback in the consumer.
5. Record boundary friction here and choose exactly one outcome: use the interfaces unchanged, open
   one finite correction, or defer further expansion.

## Acceptance

- The consumer-owned assets identify the exact Grafik source revision and verify artifact checksums.
- Both existing JSON interfaces drive one explanation through consumer-measured browser geometry.
- The consumer retains its own semantics, rendering, accessibility, and primary navigation.
- Reduced motion schedules no decorative traversal; missing or failed Grafik assets leave readable
  content and the primary action available.
- Grafik and consumer gates pass independently; neither default gate reads the sibling checkout.
- The evidence names integration friction and records one explicit next outcome without treating a
  successful demo as compatibility, adoption, or product-value proof.

## Narrow proof

```sh
cargo make artifact
cargo make test-artifact
cargo make check
```

Run the consumer repository's complete local gate separately and record its command and result here.

## Evidence audit

The earlier uncommitted completion claim is rejected. On 2026-07-19, the clean `kapsel.cloud`
`master` revision `4681fb1` contained no Grafik or WASM artifact, no Grafik attribution, no
`check:grafik-browser` script, and no committed history containing that integration. A temporary or
reverted checkout cannot establish a completed downstream boundary.

## Outcome

Do not count this packet as completed consumer evidence. The intended recorded integration is
superseded by [0012](0012-prove-one-live-kapsel-consumer.md), which must prove the boundary against
the committed live-sandbox consumer and record independently verifiable revisions in both
repositories.

## Residual risk

One downstream proof cannot establish a reusable seam, stable compatibility, broad browser quality,
or demand. It can only show whether the current renderer-neutral boundary survives one concrete
consumer without product coupling.
